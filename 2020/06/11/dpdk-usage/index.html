Posted on 2020-06-11 Edited on 2020-10-11

DPDK 编译使用记录

准备

BIOS 设置

如果 UEFI secure boot 被启用了，Linux 内核可能会禁止系统使用 UIO，请关闭相应设置或者使用 vfio-pci 内核模块，详见。

系统环境

  • Linux kernel >= 3.16
  • 内核配置
    • HUGETLBFS 大页内存
    • PROC_PAGEi_MONITOR 支持
  • GNU make
  • gcc 4.9+ / clang 3.4+
  • libc-headers(etc. glibc-devel libc6-dev)
  • libgcc
  • libstdc++
  • Library for handling NUMA(Non Uniform Memory Access)
    • numactl-devel
    • libnuma-dev
  • python 3.5+
  • kernel-headers
  • kernel-devel
  • glibc 2.7+
  • meson
  • ninjia

下载 DPDK 源文件

从官网下载源文件，解压。本文基于 19.11。

  • lib: DPDK 库文件
  • drivers: DPDK 轮询驱动源文件
  • app: DPDK 应用程序 (自动测试)源文件
  • examples: DPDK 应用例程
  • config, buildtools, mk: 框架相关的 makefile、脚本及配置文件

编译

手动编译和设置 DPDK 环境

解压并进入 DPDK 源码根目录

DPDK目标文件的格式为:

ARCH-MACHINE-EXECENV-TOOLCHAIN

其中：

  • ARCH 可以是: i686, x86_64, ppc_64, arm64
  • MACHINE 可以是: native, power8, armv8a
  • EXECENV 可以是: linux, freebsd
  • TOOLCHAIN 可以是: gcc, icc, clang

使用脚本编译和配置 DPDK 环境

为了能清晰的进行编译和配置，第一次使用时推荐使用usertools下的dpdk-setup.sh进行编译和配置。

  1. 按需进行编译，如在 x64 的 Linux 下使用 gcc 进行编译则选择15。(要和环境变量的一样)。

  2. 挂载 uio 内核模块，可选择 igb_uio 或者 vfio。推荐使用 vfio。在 qemu 虚拟机下无法使用 vfio，只能使用 igb_uio。

  3. 配置大页内存。至少 512，按照推荐的 64 会报错。

  4. 将网卡挂载到 uio 上。
